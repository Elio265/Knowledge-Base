## 打开方式

1. Obsidian 打开本仓库根目录作为 Vault。
2. 从 `obsidian-cpp-kb/00-MOC/面试知识地图.md` 或具体主题知识地图进入。
3. 使用 `[[双向链接]]` 在知识之间跳转。
4. 复习时优先看每篇笔记里的「面试高频问题」「容易踩坑的地方」「我的薄弱点」。

## 当前重点

- `01-C++/`：面向对象、STL、内存管理、智能指针、C++11、模板等。
- `02-Linux/`：进程线程、IPC、文件系统、权限、I/O 多路复用、TCP/HTTP 基础。
- `07-Database/`：MySQL 索引、事务、日志、Redis 缓存与高并发问题。
- `00-MOC/`：知识地图入口，负责学习路线和主题索引。

## 整理原则

- 目录只是存放位置，复习入口以 `00-MOC/` 为准。
- 网络知识只保留 `网络知识地图` 作为复习视角，不单独维护 Network 目录。
- 每个主题优先保留一篇主笔记，避免把零散问答拆成很多小文件。
- MOC 只放学习路线和主题入口，不堆细碎结论。
- 对面试高频、容易答错、实战常用的内容详细写；低频背景一两句话带过。
- 没有复习价值的临时材料、平台来源、窗口状态、插件产物不进仓库。

## 知识状态

| status | 含义 |
|--------|------|
| 待学习 | 标记了但还没开始 |
| 学习中 | 知道概念，缺少实践或细节 |
| 已掌握 | 可以轻松讲解、手写代码 |
| 深入理解 | 理解底层原理和设计权衡 |
| 面试重点 | 高频考点，需要反复复习 |

## AI 知识库管理规则

维护规则在隐藏配置文件 `.claude/rules.md` 中，不作为 Obsidian 笔记跳转。核心原则：

- **面试题驱动 + 会话沉淀**
- **更新优于新增，链接优于孤立**
- **高频重点详细，低频背景简略**
- **不搬运外部博客杂质，不保留平台来源痕迹**
- **先搜索已有笔记，再决定是否新增**
- **MOC 只维护主题入口和学习路线，不收集零散问答**

## 提交前检查

- 优先提交知识笔记、MOC、README 这类长期内容。
- 不要把单纯的 Obsidian 窗口布局、最近打开文件变化混入知识提交。
- 提交前检查 `git status`，确认没有误带 `.obsidian/workspace.json` 这类个人工作区状态。
