# C语言杂谈

## 一句话理解
C语言中一些容易混淆的关键概念和细节汇总，涵盖定义与声明的区别、内存对齐、宏、指针数组关系、关键字用法等基础知识。

## 核心原理

### 定义 vs 声明 vs 赋值 vs 初始化
- **定义**：创建对象，分配内存并取名
- **声明**：告知编译器名称已被占用，不分配空间
- **赋值**：给已开辟的空间写入数据
- **初始化**：变量创建时的特殊赋值，只能有一次

```cpp
int global = 10;    // 定义并初始化
void myFunction();  // 声明
int local;          // 定义
local = 20;         // 赋值
```

### 生命周期和作用域
- **生命周期**：变量从创建到销毁的时间段
- **作用域**：变量有效的代码范围
- **全局变量**：在整个程序中都可用
- **局部变量**：只在定义它的作用域内有效

### sizeof不是函数
`sizeof` 是C语言的关键字/操作符，不是函数。计算变量时可省略括号，但计算类型时不能省略。

```cpp
sizeof(val);   // 可以省略括号：sizeof val
sizeof(int);   // 类型不能省略括号
```

## 底层实现

### signed/unsigned的本质
正负数在内存中以补码形式存储，有无符号只是读取方式的差异。

```cpp
int i = -20;
unsigned int j = 10;
// i + j 的结果取决于读取方式：
// %d 输出: -10
// %u 输出: 42亿多
```

### static关键字
- **修饰全局变量**：作用域限制在声明它的源文件中
- **修饰局部变量**：生命周期扩展到整个程序执行期，只初始化一次，默认初始化为0
- **修饰函数**：作用域限制在声明它的文件内部

### 空结构体的大小
空结构体大小通常为1字节（具体依赖编译器实现），因为编译器要求每个类型都有确定的大小，以便定义变量时分配空间。

### 柔性数组（C99）
结构体最后一个元素可以是未知大小的数组，前面必须至少有一个其他成员，且只能有一个柔性数组。

```cpp
struct array {
    int length;
    int arr[];  // 柔性数组，不占用结构体大小
};
```

### union（联合体/共用体）
联合体内所有成员共享一块内存，大小为最大成员的大小。可用于判断系统大小端：

```cpp
union U {
    int a;
    char b;
};
// 若 b == 1 则为小端，否则为大端
```

### enum枚举类型
枚举类型大小由编译器决定（通常4或8字节），与枚举值有关。不需要实例化即可使用。

```cpp
enum Weekday { Monday = 1, Tuesday, Wednesday };
```

### typedef
给已有数据类型取别名，而非定义新类型。注意：不能和其他类型修饰符组合使用。

```cpp
typedef int int32;
// unsigned int32 a = 0;  // 错误
```

### 内存对齐
数据按照特定规则放在对应地址上，提高系统性能。未对齐的数据可能需要多次内存访问。

### 宏定义 #define
在预处理阶段直接进行文本替换，没有类型安全检查，存在优先级问题。

```cpp
#define M 10        // 宏作用域从定义处向后
#define STRINGIZE(x) #x     // #号将参数转为字符串
#define CONCAT(x, y) x##y   // ##连接标识符
```

### 指针和数组的关系
**指针和数组没有关系！**
- 指针大小固定为4/8字节，存放地址
- 数组大小为 `类型 * 元素个数`，存放实际数据
- 数组名类似于指针（首元素地址），但不能被重新赋值

### assert是宏不是函数
`assert` 是宏定义，只存在于debug版本，用于定位错误而非排除错误。

## 常见应用场景

1. **static**：单例模式中的静态局部变量、文件内私有函数
2. **union**：判断大小端、协议解析中不同类型的数据共享存储
3. **enum**：状态码、选项标志的定义
4. **内存对齐**：网络协议头、硬件寄存器映射的结构体设计
5. **flexible array**：消息缓冲区等变长数据结构
6. **宏定义**：日志输出、调试开关、通用代码生成

## 容易踩坑的地方

1. **sizeof对于指针和数组**：`sizeof(指针)` 恒为4/8，`sizeof(数组)` 为总字节数
2. **有符号和无符号混用**：混合运算时可能导致意外的结果（负数转为很大的正数）
3. **宏的参数副作用**：`MAX(++a, b)` 可能导致参数被多次求值
4. **static局部变量的初始化**：只初始化一次，第二次调用时跳过初始化
5. **数组名作为参数**：数组作为函数参数时会退化为指针，丢失数组大小信息
