# 智能指针

## 一句话理解
智能指针是C++中基于RAII思想设计的类模板，自动管理动态分配的内存生命周期，在指针超出作用域时自动释放资源，从根本上避免内存泄漏。

## 核心原理

### RAII（资源获取即初始化）
RAII是智能指针的核心理念：利用对象的构造函数获取资源，析构函数释放资源，确保资源的正确管理。即使发生异常，析构函数也会被自动调用，保证异常安全性。

### 引用计数（Reference Counting）
`shared_ptr` 内部维护引用计数，记录指向同一资源的智能指针数量。每次拷贝增加计数，每次析构减少计数，计数归零时释放资源。

```cpp
// shared_ptr 简化模拟：引用计数需线程安全
shared_ptr(T* ptr = nullptr) 
    : _ptr(ptr)
    , _preCount(new int(1))
    , _mtx(new std::mutex)
{}
```

## 底层实现

### unique_ptr
独占所有权，拷贝构造函数和拷贝赋值运算符被 `=delete`，只能通过移动语义转移所有权。是零开销抽象，与原始指针大小相同。

```cpp
template<class T>
class unique_ptr {
public:
    unique_ptr(T* ptr) : _ptr(ptr) {}
    ~unique_ptr() { if (_ptr) delete _ptr; }
    T& operator*() { return *_ptr; }
    T* operator->() { return _ptr; }
    // 禁止拷贝
    unique_ptr(const unique_ptr&) = delete;
    unique_ptr& operator=(const unique_ptr&) = delete;
private:
    T* _ptr;
};
```

### shared_ptr
共享所有权，内部维护：
- **资源指针**：指向管理的对象
- **引用计数指针**：指向堆上的引用计数（int*）
- **互斥量**：保证多线程环境下引用计数的线程安全

### weak_ptr
弱引用，不增加引用计数。主要用于解决 `shared_ptr` 的循环引用问题。需要使用时通过 `lock()` 获取临时的 `shared_ptr`。

### std::make_shared
一次性分配对象和引用计数的内存块，减少内存分配次数，提高性能。

## 常见应用场景

1. **unique_ptr**：工厂函数返回独占资源、PImpl惯用法、类中持有独占成员
2. **shared_ptr**：多个对象共享同一资源、缓存系统、图结构
3. **weak_ptr**：打破shared_ptr循环引用、观察者模式、缓存弱引用

## 容易踩坑的地方

1. **循环引用**：A持有B的shared_ptr，B持有A的shared_ptr，导致内存泄漏。解决：将其中一个改为weak_ptr
2. **裸指针和智能指针混用**：同一资源不要同时用裸指针和智能指针管理
3. **shared_ptr线程安全问题**：引用计数原子操作是线程安全的，但shared_ptr对象本身的修改不是线程安全的
4. **使用auto_ptr**：C++11已废弃，所有权转移时源指针悬空，隐患极大
5. **自定义删除器**：unique_ptr的自定义删除器是类型的一部分，shared_ptr不是
