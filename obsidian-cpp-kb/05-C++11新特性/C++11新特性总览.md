# C++11新特性总览

## 一句话理解
C++11是C++语言的一次重大升级，引入了自动类型推导、移动语义、Lambda表达式、智能指针、并发编程支持等现代语言特性，使C++更高效、更安全、更具表现力。

## 核心原理

### 统一列表初始化
C++11引入了花括号 `{}` 的统一初始化语法，可以用于数组、容器、结构体、类等的初始化，使代码更加清晰一致。

```cpp
int arr[] = {1, 2, 3, 4, 5};
std::vector<int> vec = {1, 2, 3, 4, 5};
std::vector<std::vector<int>> matrix = {{1, 2, 3}, {4, 5, 6}};
```

`std::initializer_list` 允许函数或构造函数接受列表形式的参数，常用于自定义容器类的初始化。

### auto类型推导
auto让编译器根据初始化表达式自动推导变量类型，简化代码编写，特别适用于处理复杂类型。

```cpp
auto i = 42;          // int
auto d = 3.14;        // double
auto lambda = [](int x){ return x * 2; };
```

注意：auto不能用于函数参数和数组类型的推导，且不能与auto&混用导致引用丢失。

### decltype
decltype用于在编译时获取表达式或变量的精确类型，特别适用于泛型编程和元编程。

```cpp
template<typename T, typename U>
auto add(T a, U b) -> decltype(a + b) {
    return a + b;
}
```

### nullptr
nullptr是C++11引入的空指针关键字，解决了NULL和0导致的二义性和类型不匹配问题。nullptr没有特定类型，不能隐式转换为其他指针类型，提高了类型安全性。

### 范围for循环
范围for循环是一种遍历容器元素的语法糖，无需手动管理迭代器或索引。

```cpp
int arr[] = {1, 2, 3, 4, 5};
for (int e : arr) {
    std::cout << e << " ";
}
```

### 右值引用与移动语义
C++11通过 `&&` 引入右值引用，区分左值和右值：

- **左值**：有持久内存地址的对象，可取地址
- **右值**：临时对象或即将销毁的对象，不可取地址

右值引用允许绑定到右值，实现移动语义（避免深拷贝）和完美转发。

```cpp
int&& rvalue = 5;  // 右值引用绑定到右值，延长其生命周期
```

**移动语义**：通过移动构造函数和移动赋值运算符转移资源所有权，避免不必要的拷贝，提升性能。

**完美转发**：使用 `std::forward` 配合模板万能引用，在函数模板中保持参数的值类别（左值/右值）和常量性。

```cpp
template<typename T>
void forwardFunction(T&& arg) {
    func(std::forward<T>(arg));
}
```

### Lambda表达式
Lambda表达式是匿名函数，语法形式为：

```cpp
[capture list](parameters) mutable -> return_type {
    // function body
}
```

- **捕获列表**：值捕获 `[x]`、引用捕获 `[&x]`、隐式值捕获 `[=]`、隐式引用捕获 `[&]`
- **mutable**：取消Lambda的const属性，允许修改值捕获的变量
- 不同Lambda表达式是不同的闭包类型，不能相互赋值

### 可变参数模板
C++11引入 `template<typename... Args>`，允许函数和类接受不定数量和类型的参数。

递归展开方式：
```cpp
void print() { std::cout << std::endl; }

template<typename T, typename... Args>
void print(const T& x, Args... args) {
    std::cout << x << " ";
    print(args...);
}
```

### =default和=delete
- `=default`：显式要求编译器生成默认版本的成员函数
- `=delete`：禁止编译器生成或使用某个函数

### 类成员变量缺省值
C++11允许在类的定义中为成员变量直接提供初始值：

```cpp
class MyClass {
public:
    int myInt = 0;
    float myFloat = 3.14f;
};
```

### 默认成员函数新增
C++11在原有6个默认成员函数基础上新增：
- 默认移动构造函数
- 默认移动赋值运算符重载

生成条件：用户没有定义析构函数、拷贝构造和拷贝赋值。

### function包装器
`std::function` 是通用的函数包装器，可以存储函数指针、函数对象、Lambda表达式等各种可调用对象，实现类型擦除，避免模板实例化导致的代码膨胀。

```cpp
std::function<void(int, int)> func;
func = sum;
func = sum_func();
func = sum_lambda;
func(10, 7);
```

### bind绑定器
`std::bind` 将函数与参数或对象绑定，生成新的可调用对象。支持占位符 `std::placeholders::_1`、`_2` 等。

```cpp
std::bind(&MyClass::hello, &obj, std::placeholders::_1, std::placeholders::_2);
```

### emplace系列接口
STL容器新增 `emplace_back`、`emplace` 等接口，在容器中就地构造元素，避免临时对象的拷贝或移动操作，提高性能。

### 线程库
C++11标准库引入了完整的线程支持：

| 组件 | 功能 |
|------|------|
| `std::thread` | 创建和管理线程 |
| `std::mutex` 系列 | 互斥量（mutex、recursive_mutex、timed_mutex、shared_mutex） |
| `std::lock_guard` / `std::unique_lock` | RAII方式的锁管理 |
| `std::atomic` | 原子操作 |
| `std::condition_variable` | 条件变量 |
| `std::async` / `std::future` | 异步编程 |

```cpp
std::thread t(threadFunction, data);
t.join();   // 等待线程结束
t.detach(); // 分离线程
```

## 底层实现

### auto的原理
auto的类型推导在编译期完成，编译器根据初始化表达式推导变量类型，不产生运行时开销。

### 右值引用的本质
右值引用本质上仍是引用，底层通过指针实现。当右值引用绑定到右值时，会延长临时对象的生命周期至引用生命周期结束。

### Lambda表达式的实现
Lambda表达式在底层被编译器转换为一个匿名的仿函数（函数对象）类，捕获的变量成为该类的成员变量。

### 移动语义的实现
移动构造函数通过"窃取"源对象的资源指针（而非深拷贝）实现高效资源转移，并将源对象置于有效但未定义的状态。

## 常见应用场景

1. **auto**：处理STL迭代器等复杂类型声明
2. **移动语义**：大型对象的传参和返回、容器操作
3. **Lambda**：STL算法回调、事件处理、临时逻辑
4. **智能指针**：自动内存管理，避免内存泄漏
5. **线程库**：多线程任务处理、并发服务器
6. **nullptr**：所有需要空指针的场景
7. **范围for**：遍历容器和数组
8. **function/bind**：回调注册、延迟调用

## 容易踩坑的地方

1. **auto引用丢失**：auto推导会忽略引用和cv限定符，需使用 `auto&` 或 `const auto&`
2. **右值引用变成左值**：在函数内部，右值引用参数本身是左值，需要 `std::move` 转移
3. **Lambda不能相互赋值**：每个Lambda表达式都是不同的闭包类型
4. **线程生命周期管理**：`std::thread` 析构时若线程仍在运行且未detach，会导致程序终止
5. **范围for中的修改**：范围for中修改元素需使用引用 `for (auto& e : vec)`
6. **emplace与构造参数**：emplace必须是构造函数的参数，而不是对象本身
