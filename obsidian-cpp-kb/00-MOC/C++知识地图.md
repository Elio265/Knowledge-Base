## 整理原则

- 每篇笔记优先回答一个核心问题，避免语法和 API 罗列。
- 容器只记选型、底层结构、复杂度和失效规则。
- 语言特性以“解决什么问题、何时使用、容易误解什么”为主。
- 数据结构用于理解 STL 和算法，不要求机械背诵全部手写细节。
