# 不定参数解析

## 一句话理解
不定参数机制允许函数接受可变数量和类型的参数，C语言通过 `stdarg.h` 宏实现（运行时），C++11通过可变参数模板实现（编译时，类型安全）。

## 核心原理

### C语言：stdarg.h宏

C语言使用 `stdarg.h` 提供的宏在运行时处理不定参数：

| 宏/类型 | 说明 |
|---------|------|
| `va_list` | 存储不定参数信息的类型 |
| `va_start(ap, last)` | 初始化va_list，指向第一个可变参数 |
| `va_arg(ap, type)` | 获取当前参数值，并将指针移动到下一个参数 |
| `va_end(ap)` | 清理va_list，释放资源 |

```cpp
#include <stdarg.h>

int sum(int n, ...) {  // n 表示后面参数的个数
    va_list args;
    va_start(args, n);
    int ret = 0;
    for (int i = 0; i < n; i++) {
        ret += va_arg(args, int);  // 需要指定类型
    }
    va_end(args);
    return ret;
}
```

**缺点**：类型不安全，需要手动管理参数类型和数量，编译器无法做类型检查。

### C语言：__VA_ARGS__ 宏

用于宏定义中表示可变参数，常用于日志宏：

```cpp
#define LOG(fmt, ...) printf("[%s : %d] " fmt "\n", __FILE__, __LINE__, ##__VA_ARGS__)
```

`##__VA_ARGS__` 中的 `##` 用于处理可变参数为空时的情况，避免末尾逗号语法错误。

### C++：可变参数模板（C++11）

类型安全的编译期不定参数处理：

```cpp
template<typename... Args>
void print(Args... args) {}
```

**展开方式一：递归展开**
```cpp
void print() {}  // 终止条件

template<typename T, typename... Args>
void print(T first, Args... args) {
    std::cout << first << " ";
    print(args...);  // 递归展开
}
```

**展开方式二：折叠表达式（C++17）**
```cpp
template<typename... Args>
void print(Args... args) {
    (std::cout << ... << args) << std::endl;
}

template<typename... Args>
int sum(Args... args) {
    return (args + ...);
}
```

**展开方式三：逗号表达式**
```cpp
template<typename... Args>
void print(Args... args) {
    int arr[] = {(print(args), 0)...};
}
```

## 底层实现

### C语言va_list原理
va_list本质上是一个指针，通过栈帧布局依次访问函数参数。参数在栈上从右向左压入，va_start根据最后一个固定参数的地址计算出第一个可变参数的地址。

### C++可变参数模板原理
可变参数模板在编译时展开，编译器为每个不同的参数组合生成独立的函数实例。没有任何运行时开销，但可能增加二进制体积。

## 常见应用场景

1. **printf/scanf函数族**：格式化输入输出
2. **日志系统**：支持可变参数的日志宏
3. **事件/回调系统**：传递不定类型和数量的参数
4. **tuple实现**：可变参数模板是实现tuple的基础
5. **工厂模式**：可变参数传递构造函数参数

## 容易踩坑的地方

1. **C语言不定参数类型不安全**：va_arg需要手动指定类型，类型错误会导致未定义行为
2. **缺少结束条件**：C语言不定参数必须知道参数个数或提供结束标志
3. **递归展开的终止条件**：必须提供空参数的函数重载作为递归终止
4. **可变参数宏中的空参数**：使用 `##__VA_ARGS__` 处理空参数避免语法错误
5. **递归展开的编译开销**：递归深度较大时增加编译时间和代码体积
