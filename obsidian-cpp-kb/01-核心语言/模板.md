# 模板

## 一句话理解
模板是C++泛型编程的基础，允许编写与类型无关的通用代码，由编译器根据实际使用自动生成特定类型的代码，实现代码复用。

## 核心原理

### 泛型编程
编写与类型无关的通用代码，模板是泛型编程的基础。通过给模板填充不同的类型参数，生成具体类型的代码。

```cpp
// 对比：需要为每种类型重载
void Swap(int& left, int& right) { ... }
void Swap(double& left, double& right) { ... }

// 模板：只需写一份
template<typename T>
void Swap(T& left, T& right) {
    T temp = left;
    left = right;
    right = temp;
}
```

### 函数模板
函数模板代表一个函数家族，在使用时根据实参类型被参数化，生成特定类型的函数版本。

**格式**：
```cpp
template<typename T1, typename T2, ...>
返回值类型 函数名(参数列表) { ... }
```

**实例化**：
- **隐式实例化**：编译器根据实参自动推导模板参数类型
- **显式实例化**：在函数名后的 `<>` 中指定类型

```cpp
Add(a1, a2);       // 隐式实例化：编译器推导T
Add<int>(a, d);    // 显式实例化：指定T为int
```

**模板参数的匹配原则**：
1. 非模板函数和同名函数模板共存时，优先调用非模板函数
2. 如果模板能生成更好的匹配，则选择模板
3. 模板函数不允许自动类型转换，普通函数可以

### 类模板
类模板的定义格式：
```cpp
template<class T1, class T2, ...>
class 类名 {
    // 类内成员定义
};
```

**注意**：
- 类模板中函数放在类外定义时，需要加模板参数列表
- 类模板名字不是真正的类，实例化的结果才是真正的类
- `Vector<int>` 才是类型，`Vector` 只是类名

```cpp
template<class T>
class Vector {
public:
    Vector();
    ~Vector();
    T& operator[](size_t pos);
};

template<class T>
Vector<T>::~Vector() { ... }
```

## 底层实现
模板本身不是函数或类，而是编译器生成代码的蓝图。在编译阶段，编译器根据模板参数进行实例化，生成对应类型的实际代码。这个过程是在编译期完成的，不产生运行时开销（零成本抽象）。

## 常见应用场景

1. **泛型容器**：vector、list、map 等STL容器
2. **泛型算法**：sort、find、copy 等STL算法
3. **类型安全的printf/format**：可变参数模板
4. **元编程**：编译期计算（如计算阶乘、类型萃取）
5. **策略模式**：通过模板参数注入不同的行为

## 容易踩坑的地方

1. **模板代码只编译不实例化**：模板定义在头文件中，但只有实例化时才会真正生成代码
2. **隐式类型转换失效**：模板函数不允许隐式类型转换，需显式实例化或强制转换
3. **模板代码膨胀**：每种类型实例化都会生成一份代码，需注意二进制体积
4. **模板参数不匹配**：多参数模板中类型不匹配导致推导失败
5. **类外定义模板语法**：类模板的成员函数在类外定义时需重复 `template` 声明
