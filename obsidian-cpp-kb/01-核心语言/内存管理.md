# 内存管理

## 一句话理解
C++内存管理涉及程序的五大内存区域分布（栈、堆、全局/静态区、常量区、代码区），以及通过new/delete进行动态内存分配和释放的完整机制。

## 核心原理

### C/C++程序内存分布

| 区域 | 存放内容 | 特点 |
|------|----------|------|
| **栈（Stack）** | 非静态局部变量、函数参数、返回值 | 向下增长，自动管理 |
| **堆（Heap）** | 动态分配的内存 | 向上增长，需手动管理 |
| **数据段（静态区）** | 全局变量、静态变量 | 程序启动分配，结束释放 |
| **代码段** | 可执行代码、只读常量 | 只读 |
| **内存映射段** | 共享动态库、共享内存 | 高效的I/O映射方式 |

### new/delete操作符

**内置类型**：
```cpp
int* p1 = new int;        // 申请单个int
int* p2 = new int(10);    // 申请并初始化为10
int* p3 = new int[10];    // 申请10个int数组
delete p1;                // 释放单个元素
delete[] p3;              // 释放数组
```

**自定义类型**：
new/delete与malloc/free的最大区别：new会调用构造函数，delete会调用析构函数，而malloc/free不会。

```cpp
class A {
public:
    A(int a = 0) : _a(a) { std::cout << "构造" << std::endl; }
    ~A() { std::cout << "析构" << std::endl; }
};

A* p = new A(10);  // 分配内存 + 调用构造函数
delete p;          // 调用析构函数 + 释放内存
```

## 底层实现

### operator new / operator delete
new和delete是操作符，其底层通过 `operator new` 和 `operator delete` 全局函数实现。

**operator new**：底层调用 `malloc` 申请空间，失败时抛出 `std::bad_alloc` 异常（而非返回NULL）。

**operator delete**：底层调用 `free` 释放空间。

### new/delete实现原理

**new单个对象**：
1. 调用 `operator new` 申请空间
2. 在申请的空间上执行构造函数

**delete单个对象**：
1. 在空间上执行析构函数
2. 调用 `operator delete` 释放空间

**new[]数组**：
1. 调用 `operator new[]`（内部调用 `operator new`）申请N个对象的空间
2. 在申请的空间上执行N次构造函数

**delete[]数组**：
1. 在空间上执行N次析构函数
2. 调用 `operator delete[]`（内部调用 `operator delete`）释放空间

### 定位new（placement-new）
在已分配的原始内存空间中调用构造函数初始化对象，通常配合内存池使用。

```cpp
A* p2 = (A*)operator new(sizeof(A));
new(p2) A(10);      // 在已分配内存上构造对象
p2->~A();           // 手动调用析构函数
operator delete(p2); // 释放内存
```

### malloc/free
- `malloc`：申请指定字节数的内存，返回 `void*`，失败返回NULL
- `free`：释放之前分配的内存
- 不对自定义类型调用构造/析构函数

## 常见应用场景

1. **new/delete**：C++标准动态内存管理
2. **placement new**：内存池实现、对象池、自定义内存管理
3. **智能指针**：自动管理new出来的对象（参见[[智能指针]]）
4. **RAII**：管理文件句柄、网络连接、互斥锁等资源

## 容易踩坑的地方

1. **new/delete不匹配**：new用delete释放（而非delete[]）导致未定义行为
2. **内存泄漏**：new后忘记delete
3. **野指针**：delete后未将指针置空
4. **malloc和new混用**：malloc分配的内存用delete释放（不会调用析构函数）
5. **placement new的显式析构**：placement new构造的对象需要显式调用析构函数
