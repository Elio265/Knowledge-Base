# 红黑树

## 一句话理解

红黑树是近似平衡的二叉搜索树。它用颜色约束控制高度，使查找、插入、删除都保持 `O(log n)`，并在读写混合场景取得较好的平衡。

## 五条性质

1. 每个节点非红即黑。
2. 根节点是黑色。
3. 空的 NIL 叶子节点是黑色。
4. 红色节点的孩子都是黑色，即不存在连续红节点。
5. 从任一节点到所有后代 NIL 叶子的路径，黑色节点数相同，称为黑高相等。

黑高相等保证各路径的黑色骨架一致；红节点不能连续，最长路径至多是在黑节点之间多插入一个红节点。因此最长路径不超过最短路径两倍，树高为 `O(log n)`。

## 修复思路

- 插入通常先插红节点：不立即改变路径黑高；若出现红父红子，再修复。
- 删除黑节点可能让某条路径少一个黑色节点，需要补偿黑高。
- 修复手段只有两类：**变色**调整黑高，**左旋/右旋**局部重排结构。旋转不改变二叉搜索树的中序有序性。

面试通常讲清“红红冲突、黑高不足、变色加旋转”即可，不必机械背删除的全部 case，除非明确要求手写。

## 与 AVL 树的取舍

| 对比 | 红黑树 | AVL 树 |
| --- | --- | --- |
| 平衡要求 | 较宽松，近似平衡 | 更严格，左右子树高度差至多 1 |
| 查找高度 | 略高 | 通常更低 |
| 插入/删除调整 | 通常更少 | 更容易触发重平衡 |
| 适合 | 读写混合 | 查询占绝对多数 |

主流标准库实现常用红黑树实现 `map/set`，但 C++ 标准只保证有序和 `O(log n)` 复杂度，不强制具体使用红黑树。

## 工程联系

- `map/set`：按 key 有序、支持 `lower_bound/upper_bound` 的范围查询。
- 调度、定时器、内存区域管理：需要按 key 有序插入、删除和查找。
- 与 [[unordered_map-unordered_set]] 的选择：需要有序/范围查询选树；只追求平均查找性能选哈希表。
