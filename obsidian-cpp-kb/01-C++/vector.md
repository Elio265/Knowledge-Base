# vector

## 一句话理解
vector 是一个动态数组容器，在连续内存上存储元素，支持随机访问和自动扩容，是 C++ 中最常用的序列容器。

## 核心原理

- vector 维护一段连续的线性内存空间，可以通过下标在 O(1) 时间内随机访问任意元素。
- 大小可动态改变，由容器自动管理内存分配与释放。
- 为减少频繁扩容，vector 会预先分配额外空间（`capacity >= size`）。
- 容量不足时通常按几何倍数增长，以保证尾部插入的均摊 O(1)；具体增长因子由标准库实现决定，C++ 标准不规定 1.5 倍或 2 倍。

### 核心成员

```cpp
vector<T> v;              // 空 vector
vector<T> v(5, 3);        // 5 个 3
vector<T> v(v2);          // 拷贝构造
vector<T> v(begin, end);  // 迭代器范围构造

v.size();      // 有效元素个数
v.capacity();  // 当前容量（已分配空间能容纳的元素数）
v.empty();     // 是否为空
v.reserve(n);  // 预留容量（不改变 size）
v.resize(n);   // 改变有效元素个数
```

### 常见接口

| 分类 | 函数 | 说明 |
|------|------|------|
| 修改 | `push_back(val)` | 尾部插入元素 |
| 修改 | `pop_back()` | 尾部删除元素 |
| 修改 | `insert(pos, val)` | 在 pos 处插入 |
| 修改 | `erase(pos)` | 删除 pos 处元素 |
| 修改 | `swap(v)` | 交换两个 vector |
| 修改 | `clear()` | 清空所有元素 |
| 访问 | `operator[]` | 下标随机访问 |
| 访问 | `front()` / `back()` | 首/尾元素引用 |
| 算法 | `find(begin, end, val)` | 非成员函数，查找元素 |

### emplace_back vs push_back

- `push_back(val)`：接收一个已存在的对象，将其拷贝或移动到容器中。
- `emplace_back(args...)`：转发构造参数，在容器尾部直接构造元素。它可能省去临时对象，但不代表始终更快。

```cpp
struct Person { Person(string n, int a); };
vector<Person> v;
v.push_back(Person("Alice", 25));    // 先构造临时对象，再移动进容器
v.emplace_back("Alice", 25);         // 直接就地构造，无临时对象
```

## 底层实现

- **连续动态数组**：底层为 `T*` 指针，指向堆上连续内存。
- **三个位置管理（常见实现）**：
  - `_start`：指向数据起始位置
  - `_finish`：指向已存储元素的下一个位置
  - `_end_of_storage`：指向已分配空间末尾
- **扩容流程**：当 `_finish == _end_of_storage` 时，申请新的更大内存 -> 拷贝/移动旧元素 -> 释放旧内存 -> 更新指针。
- **扩容复杂度**：均摊 O(1)。每次扩容代价 O(n)，但扩容次数为 O(log n)，均摊到每次插入为常数。

### 迭代器本质
`vector` 提供连续、随机访问迭代器，因此支持 `it + n`、`it - n` 等操作。迭代器在具体实现中可能是指针或封装类型，不应依赖其内部表示。

## 常见应用场景

1. **代替静态数组**：需要随机访问且大小变化的场景，如存储一组数据。
2. **作为底层容器**：`stack`、`queue`、`priority_queue` 的默认或可选底层容器。
3. **算法配合**：与 `<algorithm>` 配合进行排序、查找、二分等操作。
4. **存储大量元素**：连续内存缓存友好，遍历性能优于 list 和 map。

## 容易踩坑的地方

- **迭代器失效（最重要）**
  - **发生重新分配**：所有迭代器、指针和引用全部失效。
  - **插入但未重新分配**：插入点之前仍有效，插入点及之后失效。
  - **`erase`**：被删除元素及其之后的迭代器、指针和引用失效。
  - **正确做法**：操作后重新获取迭代器。
  ```cpp
  // 错误示例
  vector<int>::iterator it = v.begin();
  v.push_back(100);        // 可能扩容
  cout << *it;             // it 可能已失效
  
  // 正确用法：erase 后使用返回值
  auto it = v.begin();
  it = v.erase(it);        // erase 返回下一个有效迭代器
  ```

- **`reserve` vs `resize` 混淆**
  - `reserve(n)`：保证容量至少为 n，不改变 `size`，不构造或销毁元素。
  - `resize(n)`：改变 `size`；变大时构造新元素，变小时析构尾部元素，容量不足时才扩容。
- **`reserve` 不会缩小**：当参数不大于当前 `capacity` 时不会减少容量；`shrink_to_fit` 也只是非强制请求。
- **`insert` / `erase` 效率差**：非尾部位置插入/删除需要移动大量元素，复杂度 O(n)。
- **越界访问**：`operator[]` 不检查越界，越界是未定义行为（不会抛异常）。使用 `v.at(i)` 会在越界时抛 `out_of_range`。
- **扩容后迭代器失效常考**：在 `for` 循环中向 v 插入元素，遍历可能访问到失效迭代器。

- **遍历中删除元素**：`erase(it)` 后原 `it` 失效，必须接住返回的下一个有效迭代器，不能再额外 `++it`。
  ```cpp
  for (auto it = v.begin(); it != v.end();) {
      if (need_remove(*it)) it = v.erase(it);
      else ++it;
  }
  ```
  批量按值删除可用 `v.erase(std::remove(v.begin(), v.end(), value), v.end());`。

### 时间复杂度表

| 操作 | 时间复杂度 |
|------|-----------|
| 随机访问 `operator[]` | O(1) |
| 尾部插入/删除 | 均摊 O(1) |
| 中间插入/删除 | O(n) |
| 查找（线性） | O(n) |

### vector 和 list 的选择

| 容器 | 更适合 | 关键取舍 |
| --- | --- | --- |
| `vector` | 遍历、随机访问、尾部追加 | 连续内存，缓存友好，通常是默认选择 |
| `list` | 已知节点位置且频繁插入/删除 | 插删为 `O(1)`，但查找仍是 `O(n)`，节点分配和指针跳转影响缓存 |

即使存在中间插删，`vector` 也常因连续内存、预取和更少的分配而更快。只有元素移动代价高且确实已持有插删位置时，`list` 才更有优势。

## 面试高频问题

1. **vector 的扩容机制是什么？增长倍数是固定的吗？**
   - 容量不足时申请更大的连续空间，移动或拷贝元素后释放旧空间。几何增长使尾插保持均摊 O(1)，具体倍数是实现细节。
2. **`reserve(n)` 和 `resize(n)` 的区别？**
   - reserve 只保证容量不改 size；resize 改 size，变大时构造元素，容量不足时才扩容。
3. **哪些操作会导致 vector 迭代器失效？如何避免？**
   - 重新分配会使全部失效；未重新分配的插入和删除通常使操作位置及其之后失效。可预先 `reserve`，并使用修改操作返回的新迭代器。
4. **`emplace_back` 和 `push_back` 有什么区别？什么时候用哪个？**
   - 已有对象时用 `push_back`更直接；需要用构造参数直接创建元素时可用 `emplace_back`。后者可能减少临时对象，但不是无条件的性能优化。
5. **vector 的 `at()` 和 `operator[]` 的区别？**
   - `at()` 越界抛异常，`operator[]` 越界未定义行为（不检查）。
6. **遍历中如何安全地删除 vector 元素？**
   - 使用 `it = erase(it)` 接收下一个有效迭代器；批量删除可用 erase-remove 惯用法。
7. **vector 和 list 如何选择？**
   - 默认优先 vector；list 只在已知位置频繁插删、且移动元素代价较高时考虑。
