# 布隆过滤器

## 一句话理解
布隆过滤器是一种概率型数据结构，通过位图与多个哈希函数的组合，以牺牲一定的准确率为代价实现大规模数据集合的快速存在性判断。

## 核心原理

布隆过滤器（Bloom Filter）由布隆于 1970 年提出，其核心思想是：**哈希 + 位图** 的有机结合。

- 使用多个哈希函数将一个元素映射到位图中的多个位置，并将这些位置置为 1
- 查询时，检查所有映射位置是否均为 1
  - 如果**有一个为 0**：元素**一定不存在**
  - 如果**全部为 1**：元素**可能存在**（有误判风险）

### 关键特性
- **不会漏报**（false negative = 0）：元素真正存在则一定判断为存在
- **可能误报**（false positive > 0）：元素不存在但可能被误判为存在
- **不支持删除**（标准实现下）
- **空间效率极高**

### 误判率的影响因素
误判率与以下三个因素相关：
1. **位数组大小 m**：m 越大，误判率越低
2. **哈希函数个数 k**：k 过少或过多都会增加误判率，存在最优值
3. **已插入元素数量 n**：n 越大，误判率越高

最优哈希函数个数公式：k = (m/n) × ln2

## 底层实现

### 位图（底层容器）

位图通过比特位的 0/1 表示元素的存在与否。一个字节（8 bit）可以映射 8 个整数，空间效率极高。

```cpp
template<size_t N>
class bits_set {
public:
    bits_set() : _bits(N / 8 + 1) {}

    void set(size_t n) {
        size_t i = n / 8;    // 计算字节索引
        size_t j = n % 8;    // 计算位偏移
        _bits[i] |= (1 << j);
    }

    void reset(size_t n) {
        size_t i = n / 8;
        size_t j = n % 8;
        _bits[i] &= ~(1 << j);
    }

    bool test_bits(size_t n) {
        size_t i = n / 8;
        size_t j = n % 8;
        return (_bits[i] >> j) & 1;
    }
private:
    vector<char> _bits;
};
```

### 布隆过滤器实现

```cpp
template<class K = string,
         class HashFunc1 = BKDRHash,
         class HashFunc2 = APHash,
         class HashFunc3 = DJBHash>
class BloomFilter {
public:
    void set(const K& key) {
        size_t len = X * N;
        size_t hash1 = HashFunc1()(key) % len;
        _bs.set(hash1);
        size_t hash2 = HashFunc2()(key) % len;
        _bs.set(hash2);
        size_t hash3 = HashFunc3()(key) % len;
        _bs.set(hash3);
    }

    bool test(const K& key) {
        size_t len = X * N;
        size_t hash1 = HashFunc1()(key) % len;
        if (!_bs.test_bits(hash1)) return false;

        size_t hash2 = HashFunc2()(key) % len;
        if (!_bs.test_bits(hash2)) return false;

        size_t hash3 = HashFunc3()(key) % len;
        if (!_bs.test_bits(hash3)) return false;

        return true;  // 可能存在（误判）
    }
private:
    bits_set<X * N> _bs;
};
```

### 常见哈希函数

#### BKDRHash
由 Brian Kernighan 和 Dennis Ritchie 提出，适合字符串哈希：
```cpp
struct BKDRHash {
    size_t operator()(const string& s) {
        size_t value = 0;
        for (auto ch : s) {
            value *= 31;
            value += ch;
        }
        return value;
    }
};
```

#### APHash (Adler-32 变体)
通过位运算和异或操作组合字符：
```cpp
struct APHash {
    size_t operator()(const string& s) {
        size_t hash = 0;
        for (long i = 0; i < (long)s.size(); i++) {
            if ((i & 1) == 0)
                hash ^= ((hash << 7) ^ s[i] ^ (hash >> 3));
            else
                hash ^= (~((hash << 11) ^ s[i] ^ (hash >> 5)));
        }
        return hash;
    }
};
```

#### DJBHash
Daniel J. Bernstein 于 1991 年设计，使用常数 5381 作为初始哈希值：
```cpp
struct DJBHash {
    size_t operator()(const string& s) {
        size_t hash = 5381;
        for (auto ch : s) {
            hash = ((hash << 5) + hash) + ch;
        }
        return hash;
    }
};
```

### 空间效率对比

以"40 亿个无符号整数判存在"问题为例：
| 方式 | 空间占用 | 说明 |
|-----|---------|------|
| 整型数组直接存储 | ~16 GB | 40 亿 × 4 字节 |
| 位图 | ~512 MB | 16 GB ÷ 32 比特/整型 |
| 布隆过滤器 | 约位图级别 | 加上哈希函数的额外开销很小 |

## 常见应用场景
- **网络爬虫**：快速排除已经爬取的 URL，避免重复爬取
- **缓存系统**：判断数据是否在缓存中，减少数据库访问（防止缓存穿透）
- **防止重复提交**：Web 应用中防止用户重复提交表单
- **垃圾邮件过滤**：快速排除已知垃圾邮件地址
- **数据库查询优化**：快速判断记录是否在数据库中，减少昂贵的磁盘查询
- **用户昵称/用户名判重**：快速筛选可用昵称（误判的少量冲突可二次验证）

特别说明：布隆过滤器适合**能容忍少量误判**的场景。判断"不存在"是确定的，判断"存在"才可能误判。因此适合做快速筛选的"第一道关"。

## 容易踩坑的地方
1. **不支持删除**：标准布隆过滤器无法删除元素，因为多个哈希映射可能共享同一个位，直接清零会导致误判率增加。需要删除的场景必须使用计数布隆过滤器（Counting Bloom Filter）。
2. **哈希函数的选择**：哈希函数的质量直接影响误判率。需要选择分布均匀、冲突率低的哈希函数，且不同哈希函数之间应该独立。
3. **位数组大小的上限**：位数组索引大小受限于 `size_t` 范围。对于超大规模数据集，需要使用分片布隆过滤器。
4. **不能替代精确过滤**：布隆过滤器只能做概率性判断，实际系统中通常作为第一层快速过滤，后面再接精确查询（如数据库或哈希表）。
5. **误判率随插入数量增加而升高**：设计时需要预留足够的位数组空间。
6. **不适合整型有范围限制的场景**：位图适用于整数范围内的元素。对于非整型元素（如字符串），需要先通过哈希函数转换。
