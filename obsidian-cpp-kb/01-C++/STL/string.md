# string

## 一句话理解

`std::string` 是管理字符序列的 RAII 类型：自动管理内存、保存长度、支持拼接和查找。除 C 接口交互外，工程中通常优先于 C 风格字符串。

## `string` 与 C 风格字符串

| 对比 | `std::string` | `char*` / `char[]` |
| --- | --- | --- |
| 本质 | 拥有字符序列的对象 | `char*` 是指针，`char[]` 是数组 |
| 长度 | 自身保存，可 `O(1)` 获取 | 以第一个 `\0` 结束，常靠 `strlen` 扫描 |
| 内存 | 自动管理 | 容量、复制和生命周期通常需手动处理 |
| 内容 | 可包含嵌入的 `\0` | C 字符串遇到第一个 `\0` 即结束 |

`std::string` 是 `std::basic_string<char>` 的别名，字符连续存储，支持随机访问和 STL 风格迭代器。

## 常用接口与复杂度

| 操作 | 要点 |
| --- | --- |
| `size()` / `length()` | 等价，`O(1)`；嵌入的 `\0` 也计入长度 |
| `capacity()` | 已分配的可用容量 |
| `reserve(n)` | 预留容量，不改变 `size()` |
| `resize(n, c)` | 改变长度；扩大时填充字符，默认填 `\0` |
| `+=` / `append()` | 尾部追加 |
| `find()` | 查找失败返回 `npos`，不要和 `-1` 比较 |
| `substr()` | 返回新的 `string`，会复制子串 |
| `c_str()` | 返回以 `\0` 结尾的 `const char*`，用于 C API |

`strlen(s.c_str())` 是 `O(n)`，且会在第一个 `\0` 停止；它不能代替 `s.size()`。

## SSO 与拼接性能

**SSO（小字符串优化）**：常见实现会把短内容直接存入 `string` 对象内部，避免堆分配；超过内部容量后才转为堆内存。它不是标准保证，不能依赖特定阈值或对象布局。

大量小片段拼接时，容量不足会反复重分配和复制。能预估最终长度时先 `reserve()`：

```cpp
std::string out;
out.reserve(total_size);
for (const auto& part : parts) {
    out += part;
}
```

`operator+` 在简单表达式中可能被移动或优化，但复杂链式拼接仍可能产生临时对象；批量拼接优先 `reserve + append/+=`。

## `c_str()`、引用与迭代器失效

`c_str()` 返回的是借用指针，只要 `string` 未被销毁、移动或发生可能重分配的修改，就可使用。追加、插入、替换、`reserve`、缩容等操作后都可能失效。

最稳的工程规则：**不要跨越对字符串的修改保存 `c_str()`、`data()`、引用、指针或迭代器。**

```cpp
const char* p = s.c_str();
s += "x";
// p 之后不要使用
```

`operator[]` 不检查越界；需要边界检查时用 `at()`，越界会抛 `std::out_of_range`。

## `string_view`

`std::string_view`（C++17）是非拥有的“指针 + 长度”视图，不复制、不分配，适合只读传参和轻量子串操作。

```cpp
std::string_view name = "alice";
```

它不延长底层字符的生命周期。原 `string` 销毁、移动、扩容或修改后，`string_view` 可能悬空；因此适合短期使用，不要随意长期保存。

## 易错点

- `clear()` 只清空内容，不保证释放容量；`shrink_to_fit()` 也是非强制请求。
- `new char[]` 与 `std::string` 的所有权不能混淆；与 C API 交互时，传 `c_str()` 仅表示借用。
- COW（写时复制）是历史实现策略；现代代码不要依赖它，`std::string` 拷贝按独立值语义理解即可。
- `char*` 不一定指向可修改或以 `\0` 结尾的字符串；`const char*` 也不表达所有权。
