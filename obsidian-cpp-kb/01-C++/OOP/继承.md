# 继承

## 面试主线

继承表达的是 **is-a** 关系：`Derived` 可以在需要 `Base` 的地方使用。工程中优先组合（has-a）；需要替换性或运行时多态时再使用公有继承。

## 访问控制与向上转型

```cpp
class Derived : public Base {};
```

| 继承方式 | 基类 `public` | 基类 `protected` | 基类 `private` |
| --- | --- | --- | --- |
| `public` | `public` | `protected` | 派生类不可直接访问 |
| `protected` | `protected` | `protected` | 派生类不可直接访问 |
| `private` | `private` | `private` | 派生类不可直接访问 |

最常用的是 `public` 继承，它保留基类对外接口，因此派生类可以替代基类。

```cpp
Derived d;
Base* p = &d;  // 向上转型，不发生切片
Base& r = d;  // 向上转型，不发生切片
Base b = d;   // 按值复制，只保留 Base 部分：对象切片
```

基类对象不能安全地直接当作派生类对象使用。向下转型应保证真实对象类型匹配；多态体系中优先用 `dynamic_cast` 检查。

## 构造、析构与默认成员函数

构造顺序：**基类 -> 成员对象（按声明顺序）-> 派生类自身**。析构顺序完全相反。

派生类依赖基类部分，所以必须先构造基类；派生类清理时，基类部分仍然有效，所以必须先析构派生类。

编译器生成派生类的拷贝构造、拷贝赋值等默认成员函数时，也会依次处理基类子对象和成员对象。

## 同名隐藏与重写

派生类只要声明同名成员函数，就会隐藏基类的**全部同名重载**，即使参数不同。

```cpp
class Base {
public:
    void f(int);
    void f(double);
};

class Derived : public Base {
public:
    using Base::f;  // 重新暴露基类重载
    void f(const char*);
};
```

- 同名即可隐藏；它不是重写。
- 重写要求基类函数是 `virtual`，且函数签名兼容；派生类应写 `override`。

## 虚析构与构造/析构中的虚调用

可能通过基类指针删除派生对象时，基类析构函数必须是虚函数：

```cpp
class Base {
public:
    virtual ~Base() = default;
};
```

否则 `delete Base*` 指向的派生对象时，派生类析构逻辑可能不执行，属于未定义行为，常见后果是资源泄漏。

构造或析构函数中调用虚函数，不会分派到尚未构造完成或已析构完成的派生层：在 `Base` 构造/析构期间只调用 `Base` 版本。派生类状态不完整时访问它很危险，因此不要依赖这类虚调用；调用纯虚函数通常会出错。

## 多继承与虚继承

菱形继承会让公共基类出现两份，造成数据冗余和成员访问二义性：

```cpp
class A {};
class B : virtual public A {};
class C : virtual public A {};
class D : public B, public C {};  // D 中只有一份 A
```

`virtual` 继承让 `D` 只保留一份 `A`；这份虚基类由最派生类 `D` 负责构造。编译器通常会通过偏移表等方式定位虚基类，但具体对象布局属于 ABI 实现细节，不是标准规则。

## 易错点

- `Base*`/`Base&` 绑定派生对象是向上转型，不是切片；按值赋给 `Base` 对象才切片。
- 基类 `private` 成员仍属于派生对象的一部分，但派生类不能直接访问。
- 不要把“同名隐藏”说成重写；重写只针对虚函数。
- 公有继承表达 is-a；仅为复用实现时，通常组合更清晰。
