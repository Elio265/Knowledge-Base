# 类和对象

## 一句话理解

C++中的类是一种用户自定义的数据类型，它将数据和操作数据的方法封装在一起；对象是类的实例化，占用实际的内存空间。

## 核心原理

### 类的定义

C++中类使用 `class` 关键字定义，类体由成员变量（属性）和成员函数（方法）组成。与C语言的结构体不同，C++的类中不仅可以定义变量，还可以定义函数。

```cpp
class className
{
    // 成员变量和成员函数
};  // 分号不能省略
```

### 访问限定符与封装

| 访问限定符 | 作用 |
|-----------|------|
| public | 类外可以直接访问 |
| protected | 类外不能直接访问，但在派生类中可访问 |
| private | 类外不能直接访问（默认访问权限） |

**封装**：将数据和操作数据的方法有机结合，隐藏对象的属性和实现细节，仅对外公开接口。C++通过类将数据与方法结合，通过访问权限控制接口的可见性。

**注意**：`class` 的默认访问权限为 `private`，`struct` 为 `public`（兼容C语言）。访问限定符只在编译时有意义，数据映射到内存后没有任何区别。

### 类的作用域与实例化

- 类定义了一个新的作用域，使用 `::` 作用域操作符指明成员所属的类域
- 类是一个模型，定义出类并没有分配实际的内存空间
- 实例化出的对象占用实际的物理空间
- 类的大小为成员变量之和（需考虑内存对齐），空类大小为1字节（用于唯一标识该对象）

### this指针

C++编译器给每个**非静态成员函数**增加了一个隐藏的指针参数 `this`，指向当前对象（函数运行时调用该函数的对象）。

- `this` 指针类型：`类类型* const`，不能给 `this` 赋值
- 只能在成员函数内部使用
- `this` 是成员函数的形参，对象中不存储 `this` 指针
- 一般由编译器通过 `ecx` 寄存器自动传递

### 构造函数

构造函数是特殊的成员函数，名字与类名相同，创建对象时由编译器自动调用，用于初始化对象。

**特征**：
- 函数名与类名相同，无返回值
- 对象实例化时自动调用
- 可以重载

```cpp
class Date {
public:
    Date() {}                          // 无参构造函数
    Date(int year, int month, int day) // 带参构造函数
        : _year(year), _month(month), _day(day) {}
private:
    int _year;
    int _month;
    int _day;
};
```

**默认构造函数**：无参构造函数、全缺省构造函数、编译器自动生成的构造函数，都可以称为默认构造函数。一个类只能有一个默认构造函数。

**初始化列表**：以一个冒号开始，以逗号分隔的数据成员列表。以下成员必须在初始化列表中初始化：
- 引用成员变量
- `const` 成员变量
- 自定义类型成员（该类没有默认构造函数时）

**注意事项**：
- 成员变量在类中的声明次序就是初始化列表中的初始化顺序，与初始化列表中的先后次序无关
- 构造函数体中的语句是赋初值，不是初始化（初始化只能一次）
- C++11 支持在类中声明时给内置类型成员默认值（缺省值）

**explicit关键字**：修饰构造函数，禁止隐式类型转换。

### 析构函数

析构函数在对象销毁时自动调用，完成资源的清理工作。函数名是在类名前加 `~`。

- 无参数、无返回值
- 一个类只能有一个析构函数，不能重载
- 若未显式定义，系统自动生成默认析构函数
- 对象生命周期结束时自动调用

如果类中没有申请资源（如动态内存、文件句柄等），可以不写析构函数；有资源申请时必须显式定义。

### 拷贝构造函数

拷贝构造函数是构造函数的重载，参数是对本类类型对象的引用（常用 `const` 修饰），用已存在的对象创建新对象时调用。

**特征**：
- 参数必须使用引用，否则会引发无穷递归（传值会调用拷贝构造函数）
- 未显式定义时，编译器会生成默认拷贝构造函数（浅拷贝/值拷贝）

**典型调用场景**：
1. 使用已存在对象创建新对象
2. 函数参数类型为类类型对象（传值）
3. 函数返回值类型为类类型对象（传值）

**注意**：涉及资源申请时，必须显式定义拷贝构造函数进行深拷贝。

### 赋值运算符重载

C++通过 `operator` 关键字实现运算符重载，增强代码可读性。

**规则**：
- 不能创建新的操作符（如 `operator@`）
- 重载操作符必须有一个类类型参数
- 内置类型的运算符含义不能改变
- `.*` `::` `sizeof` `?:` `.` 这5个运算符不能重载

**赋值运算符格式**：
```cpp
T& operator=(const T& t) {
    if (this != &t) {  // 检测自赋值
        // 赋值操作
    }
    return *this;  // 支持连续赋值
}
```

赋值运算符只能重载为成员函数，不能重载为全局函数。

**前置++和后置++重载**：
```cpp
// 前置++：返回+1之后的结果（引用返回）
Date& operator++() { _day += 1; return *this; }
// 后置++：多增int参数，返回+1之前的值（值返回）
Date operator++(int) { Date tmp(*this); _day += 1; return tmp; }
```

### 取地址运算符重载

`operator&` 和 `const operator&` 一般不需要重载，编译器默认生成即可。特殊场景（如想隐藏真实地址）才需要重写。

### static成员

用 `static` 修饰的类成员称为静态成员。

- 静态成员为所有类对象共享，存放在静态区
- 静态成员变量必须在**类外定义和初始化**，定义时不加 `static` 关键字
- 可通过 `类名::静态成员` 或 `对象.静态成员` 访问
- 静态成员函数没有隐藏的 `this` 指针，**不能访问任何非静态成员**
- 受访问限定符的限制

### 友元

友元提供了一种突破封装的方式，但会增加耦合度，不宜多用。

**友元函数**：定义在类外部的普通函数，不属于任何类，但可以访问类的私有成员。在类内部声明时加 `friend` 关键字。

**友元类**：友元类的所有成员函数都可以是另一个类的友元函数。
- 友元关系是单向的，不具有交换性
- 友元关系不能传递
- 友元关系不能继承

### 内部类

如果一个类定义在另一个类的内部，这个类称为内部类。
- 内部类是外部类的友元类，可以通过外部类的对象参数访问外部类中的所有成员
- 内部类可以直接访问外部类中的 `static` 成员，不需要外部类的对象/类名
- `sizeof(外部类) = 外部类`，和内部类无关

### 引用

引用是变量的别名，在C++中广泛用于对象传递。

- 引用在定义时必须初始化
- 一个变量可以有多个引用
- 引用一旦引用一个实体，就不能再引用其他实体
- 权限可以缩小和平移，但**不能扩大**（如 `const int` 不能赋给 `int&`）

```cpp
const int a = 10;
// int& ra = a;      // 错误：权限扩大
const int& ra = a;    // 正确：权限平移
const int& b = 10;    // 正确：常量引用
```

**引用与指针的区别**：
| 引用 | 指针 |
|------|------|
| 变量的别名 | 存储变量地址 |
| 必须初始化 | 可以不初始化 |
| 不能改变指向 | 可以改变指向 |
| 没有NULL引用 | 有NULL指针 |
| `sizeof` 结果为引用类型大小 | `sizeof` 结果为地址空间大小 |
| 自加 = 实体+1 | 自加 = 偏移一个类型大小 |

### 内联函数

以 `inline` 修饰的函数叫内联函数，编译时在调用处展开，没有函数调用建立栈帧的开销。

- 是一种**以空间换时间**的做法
- `inline` 对编译器只是一个建议，编译器可能忽略（如函数体过大、递归等）
- 任何在类中声明的函数自动成为内联函数
- 不建议声明和定义分离，否则会导致链接错误

## 底层实现

### 对象的内存布局

- 非静态成员变量按声明顺序存储在对象中（考虑内存对齐）
- 成员函数不占用对象空间，存储在代码段
- 静态成员变量存储在静态区，不属于任何对象
- 虚函数存在时，对象头部会有一个虚表指针（vptr）
- 空类大小为1字节

### 默认成员函数的编译器生成机制

当用户没有显式定义时，编译器会在需要时生成默认版本。但编译器生成的默认构造函数只对自定义类型成员调用其默认构造函数，对内置类型**不初始化**（C++11可通过缺省值改善）。

## 常见应用场景

- **封装数据和行为**：将相关的数据和操作放在一个类中，如 `Date`、`Student` 等实体类
- **资源管理（RAII）**：在构造函数中获取资源，析构函数中释放资源
- **类型转换控制**：使用 `explicit` 防止隐式转换
- **全局状态管理**：通过 `static` 成员共享数据
- **运算符重载**：自定义类型的算术、比较、输入输出等操作

## 容易踩坑的地方

1. **构造函数与默认构造函数**：`Date d();` 会被编译器理解为函数声明而非创建对象
2. **浅拷贝问题**：类中有指针成员时，默认拷贝构造函数只做浅拷贝，会导致多个对象指向同一块内存，析构时重复释放
3. **初始化列表顺序**：初始化顺序由成员变量在类中的声明次序决定，而非初始化列表中的顺序
4. **自赋值检测**：赋值运算符重载中必须先检测是否自赋值，否则可能导致内存错误
5. **引用返回局部变量**：函数返回时如果返回对象已销毁，不能使用引用返回
6. **引用权限扩大**：`const` 对象不能赋给非 `const` 引用
7. **友元破坏封装**：友元增加了耦合度，不宜多用

## 面试高频问题

1. 为什么拷贝构造函数参数必须用引用？—— 传值会调用拷贝构造函数，引发无穷递归
2. 空类占多少字节？为什么？—— 1字节，用于唯一标识对象
3. 什么是初始化列表？哪些成员必须在初始化列表中初始化？—— 引用、const成员、无默认构造函数的自定义类型成员
4. `explicit` 关键字的作用？—— 禁止构造函数隐式类型转换
5. `static` 成员函数和普通成员函数的区别？—— 没有this指针，不能访问非静态成员
6. 浅拷贝和深拷贝的区别？—— 浅拷贝按字节复制，深拷贝复制指针指向的内容
7. 引用和指针的区别？—— 见上表
8. 内联函数的优缺点？—— 以空间换时间，减少调用开销
9. `sizeof` 一个包含虚函数的类的对象？—— 包含虚表指针大小
