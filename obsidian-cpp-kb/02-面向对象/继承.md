# 继承

## 一句话理解

继承是面向对象程序设计中建立类之间层次结构的机制，允许派生类（子类）复用基类（父类）的属性和方法，并在此基础上进行扩展和修改，实现代码复用。

## 核心原理

### 继承的概念

继承体现了面向对象程序设计的层次结构：**父类表示一般化概念，子类表示更具体的概念**。通过继承，可以在不改变原有类的情况下扩展功能，减少重复代码。

### 继承的定义与访问级别

```cpp
class 派生类名 : 访问级别 基类名 {
    // 派生类的成员
};
```

访问级别可以是 `public`、`protected` 或 `private`，默认（类定义中）为 `private` 继承。

**继承方式对基类成员的访问影响**：

| 基类成员/继承方式 | public继承 | protected继承 | private继承 |
|-------------------|-----------|--------------|------------|
| 基类 public 成员 | 派生类 public | 派生类 protected | 派生类 private |
| 基类 protected 成员 | 派生类 protected | 派生类 protected | 派生类 private |
| 基类 private 成员 | 不可见 | 不可见 | 不可见 |

**总结**：
- 基类的 `private` 成员在派生类中无论以什么方式继承都是不可见的
- `public` 继承是最常用的方式，保持基类接口不变
- `protected` 和 `private` 继承很少使用

### 基类和派生类对象赋值转换（切片）

派生类对象可以赋值给基类的对象、基类的指针和基类的引用（称为切片/切割）。

```cpp
Derived d;
Base b = d;       // 派生类 -> 基类对象（切片）
Base* pb = &d;    // 派生类 -> 基类指针
Base& rb = d;     // 派生类 -> 基类引用
```

**注意**：
- 基类对象**不能直接**赋值给派生类对象
- 基类的指针可以通过强制类型转换赋值给派生类的指针，但只有基类指针确实指向派生类对象时才是安全的
- 对于多态类型，可以使用 `dynamic_cast` 进行安全的向下转换

### 继承中的作用域（隐藏/重定义）

基类和派生类有独立的作用域。当子类和父类有同名成员时，子类成员会**屏蔽**父类对同名成员的直接访问，这种情况称为**隐藏**或**重定义**。

```cpp
class Base {
public:
    void func() { cout << "Base" << endl; }
};
class Derived : public Base {
public:
    void func() { cout << "Derived" << endl; } // 隐藏了 Base::func
    void test() {
        func();       // 调用 Derived::func
        Base::func(); // 显式调用 Base::func
    }
};
```

**注意**：成员函数的隐藏只需函数名相同即可构成隐藏，即使参数列表不同也会隐藏父类中的同名函数。

### 派生类的默认成员函数

如果派生类没有显式定义6个默认成员函数，编译器会自动生成，规则如下：

| 成员函数 | 默认生成规则 |
|---------|------------|
| 构造函数 | 必须调用基类构造函数初始化基类成员 |
| 拷贝构造函数 | 必须调用基类拷贝构造函数 |
| 赋值运算符 | 必须调用基类赋值运算符 |
| 析构函数 | 调用完成后自动调用基类析构函数 |

**构造和析构顺序**：
- 构造：先调用基类构造函数，再调用派生类构造函数
- 析构：先调用派生类析构函数，再调用基类析构函数

### 继承、友元与静态成员

- **友元不能被继承**：基类的友元不能访问子类的私有和保护成员
- **静态成员共享**：基类中的静态成员在整个继承体系中只有一个实例，所有派生类共享同一个静态成员

### 继承与组合的选择

| 特性 | 公有继承（is-a） | 对象组合（has-a） |
|------|----------------|-----------------|
| 关系 | 派生类对象也是基类对象 | 每个对象中包含另一个对象 |
| 复用方式 | 白箱复用（内部细节可见） | 黑箱复用（内部细节不可见） |
| 耦合度 | 高 | 低 |
| 封装性 | 可能破坏基类封装 | 良好的封装性 |

**建议**：优先使用对象组合而不是类继承，只有在需要实现多态时才使用继承。

## 底层实现

### 单继承的对象模型

派生类对象的内存布局由两部分组成：
1. **基类部分**：包含基类的所有成员（包括基类的虚表指针）
2. **派生类部分**：派生类自己的成员

### 多继承的对象模型

当派生类继承多个基类时，对象中包含多个基类子对象，每个基类子对象包含各自的成员。

### 菱形继承与虚拟继承

**菱形继承**：一个派生类同时继承了两个直接或间接基类，而这两个基类又共同继承自同一个基类，形成菱形结构。

```
    Animal
   /      \
 Cat     Dog
   \      /
    Pet
```

**问题**：派生类（Pet）对象中包含两份基类（Animal）的成员副本，造成资源浪费和二义性。

**解决方案**：菱形虚拟继承，使用 `virtual` 关键字：

```cpp
class Animal { /* ... */ };
class Cat : virtual public Animal { /* ... */ };
class Dog : virtual public Animal { /* ... */ };
class Pet : public Cat, public Dog { /* ... */ };
```

虚拟继承通过**虚基类指针（vbptr）**和**虚基类表（vbtable）**机制，使得派生类中只共享一份基类成员，解决了数据冗余和二义性问题。

## 常见应用场景

- **代码复用**：多个类有共同属性和方法时，抽象成基类
- **层次化设计**：如动物->哺乳动物->狗/猫
- **多态实现**：继承是实现运行时多态的必要条件
- **框架扩展**：通过继承框架提供的基类来扩展功能

## 容易踩坑的地方

1. **切片问题**：派生类对象赋值给基类对象时，派生类独有的成员会被"切掉"
2. **隐藏误认为重写**：非虚函数在派生类中同名会构成隐藏而非重写，容易导致非预期的函数调用
3. **菱形继承的数据冗余**：不使用虚拟继承会导致基类成员在派生类中存在多份副本
4. **友元不可继承**：基类的友元不能自动访问派生类的私有成员
5. **构造函数调用歧义**：多继承时需要在初始化列表中显式调用每个基类的构造函数
6. **强制类型转换风险**：基类指针强转为派生类指针时，如果实际对象不是该派生类类型，会导致越界访问

## 面试高频问题

1. 什么是切片（slicing）？—— 派生类对象赋值给基类对象时，派生类特有的成员被切掉
2. 隐藏和重写的区别？—— 隐藏只需同名函数即可（可不同参数），重写需要虚函数且函数签名相同
3. 菱形继承的问题及解决方案？—— 数据冗余和二义性，使用虚拟继承解决
4. `public`/`protected`/`private` 继承的区别？—— 见上表
5. 派生类对象的构造和析构顺序？—— 先构造基类再构造派生类，先析构派生类再析构基类
6. 为什么基类析构函数通常声明为虚函数？—— 确保通过基类指针删除派生类对象时调用正确的析构函数
7. 继承和组合如何选择？—— 继承是 is-a 关系，组合是 has-a 关系；优先使用组合
