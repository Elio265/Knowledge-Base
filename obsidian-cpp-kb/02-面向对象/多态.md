# 多态

## 一句话理解

多态是指同一操作作用于不同的对象时产生不同的执行结果，在C++中通过虚函数机制实现运行时动态绑定。

## 核心原理

### 多态的概念

多态描述了**不同对象对同一消息作出不同响应**的能力。例如，动物类有一个"叫声"方法，狗实现为"汪汪汪"，猫实现为"喵喵喵"，调用相同的方法得到不同的结果。

多态性使程序更加灵活和可扩展，可以通过添加新的子类扩展功能而无需修改原有代码。

### 多态的构成条件

1. **继承关系**：通过基类指针或引用引用派生类对象
2. **虚函数**：基类中使用 `virtual` 关键字声明函数为虚函数，派生类重写该函数
3. **动态绑定**：运行时根据对象实际类型确定调用的函数
4. **基类指针或引用**：通过统一的接口调用不同子类的实现

```cpp
class Person {
public:
    virtual void action() { cout << "Person" << endl; }
};
class Student : public Person {
public:
    virtual void action() override { cout << "Student" << endl; }
};
void test() {
    Person* p1 = new Person();
    Person* p2 = new Student();
    p1->action();  // Person
    p2->action();  // Student（多态）
}
```

### 虚函数

在基类中声明为 `virtual` 的成员函数，在派生类中可以进行重写。当通过基类指针或引用调用虚函数时，根据指针或引用所指向的对象的实际类型来决定调用哪个虚函数。

**注意**：虚函数不能是静态函数。因为静态函数与类绑定，不与对象实例绑定，无法通过虚表实现动态绑定。

### 虚函数的重写（覆盖）

重写是指派生类重新定义基类中同名的虚函数。

**重写条件**：
1. 基类函数必须是虚函数（有 `virtual` 关键字）
2. 派生类函数与基类虚函数的**名称、参数列表、返回类型**必须相同
3. 派生类的 `virtual` 关键字可选（但建议加上）

**重写的两个例外**：

1. **协变**：派生类重写基类虚函数时，返回值可以是基类返回值类型的派生类型（指针或引用）
   ```cpp
   class Base {
   public:
       virtual Base* create() { return new Base(); }
   };
   class Derived : public Base {
   public:
       virtual Derived* create() override { return new Derived(); } // 协变
   };
   ```

2. **析构函数重写**：如果基类的析构函数为虚函数，派生类的析构函数（不论是否加 `virtual`）都会与基类析构函数构成重写。编译器对析构函数名做了特殊处理，统一处理为 `destructor`。

### override 与 final 关键字（C++11）

- **`final`**：修饰虚函数，表示该虚函数不能再被派生类重写
  ```cpp
  virtual void action() final { /* ... */ }
  ```

- **`override`**：检查派生类虚函数是否重写了基类虚函数，如果没有则编译报错
  ```cpp
  void action() override { /* ... */ }
  ```

### 抽象类（纯虚函数）

纯虚函数是在基类中声明但不定义的虚函数，语法为在函数声明后加 `= 0`。包含纯虚函数的类称为抽象类，不能实例化对象。

```cpp
class Shape {
public:
    virtual void draw() = 0; // 纯虚函数
};
class Circle : public Shape {
public:
    void draw() override { cout << "绘制圆形" << endl; }
};
```

派生类必须重写所有纯虚函数才能实例化对象，否则派生类也是抽象类。

### 重载、覆盖（重写）、隐藏（重定义）的对比

| 特性 | 重载 | 覆盖（重写） | 隐藏（重定义） |
|------|------|-------------|--------------|
| 作用域 | 同一作用域 | 基类和派生类 | 基类和派生类 |
| 函数名 | 相同 | 相同 | 相同 |
| 参数列表 | 不同 | 相同 | 可不同 |
| virtual | 不需要 | 基类必须有 virtual | 不需要 |
| 行为 | 编译器根据参数选择 | 运行时根据对象类型选择 | 根据声明类型选择 |

## 底层实现

### 虚函数表（vtable）

虚函数表是C++实现多态的核心机制。每个定义了虚函数的类都有一个对应的虚函数表，用于存储该类的虚函数地址。

- vtable 是一个由**函数指针构成的数组**，每个指针指向一个虚函数的实现
- 每个对象中有一个**虚表指针（vptr）**，指向其对应类的 vtable
- vptr 通常是对象的第一个成员（由编译器决定位置）
- vtable 本身在编译阶段由编译器生成，通常存储在**代码段**中

### 派生类虚表的生成过程

1. 将基类中的虚表内容拷贝一份到派生类的虚表中
2. 如果派生类重写了基类的某个虚函数，用派生类的虚函数**覆盖**虚表中对应的函数指针
3. 派生类新增加的虚函数按声明顺序添加到虚表的最后

**关键理解**：虚函数的重写在底层上就是虚表指针的覆盖，所以"覆盖"是原理层的叫法，"重写"是语法层的叫法。

### 动态绑定过程

```
对象 ----> [vptr] ----> [vtable] ----> [虚函数地址]
```

当通过基类指针调用虚函数时：
1. 通过对象的 vptr 找到对应的 vtable
2. 在 vtable 中查找该虚函数的指针
3. 通过函数指针调用实际的函数实现

这个过程在运行时完成，实现了动态绑定。

## 常见应用场景

- **接口设计**：通过基类定义接口（纯虚函数），派生类提供具体实现
- **工厂模式**：根据条件创建不同派生类对象，通过基类指针统一使用
- **策略模式**：不同派生类实现不同的算法策略
- **框架扩展点**：框架定义虚函数接口，用户通过派生类定制行为
- **资源管理**：虚析构函数确保正确释放资源

## 容易踩坑的地方

1. **构造函数中调用虚函数**：构造函数中调用虚函数不会产生多态，因为派生类此时尚未构造完成，调用的是当前正在构造的类版本
2. **析构函数中调用虚函数**：同样不会产生多态，派生类部分已经销毁
3. **虚函数默认参数**：虚函数的默认参数是静态绑定的（根据指针的静态类型），而不是动态绑定
4. **遗漏 override**：派生类中重写虚函数时忘记写 `override`，如果签名不匹配会变成隐藏而非重写
5. **普通析构函数导致内存泄漏**：通过基类指针删除派生类对象时，如果析构函数不是虚函数，只会调用基类析构函数，派生类资源无法释放
6. **虚函数性能开销**：虚函数调用需要间接寻址（通过vtable），比普通函数调用多一次内存访问
7. **静态函数不能是虚函数**：静态函数不属于对象实例，无法通过 vptr 机制实现多态

## 面试高频问题

1. 多态的实现原理？—— 虚函数表（vtable）+ 虚表指针（vptr）+ 动态绑定
2. 虚函数表存在哪里？—— 代码段（常量区），由编译器在编译时生成
3. 派生类的虚表如何生成？—— 拷贝基类虚表 -> 覆盖重写的虚函数 -> 添加新的虚函数
4. 构造函数和析构函数中能调用虚函数吗？—— 可以调用但不会多态
5. 为什么基类析构函数要声明为虚函数？—— 确保通过基类指针删除派生类对象时调用正确的析构链
6. 静态函数可以是虚函数吗？为什么？—— 不可以，静态函数没有 this 指针，不依赖对象实例
7. 哪些函数不能是虚函数？—— 构造函数、静态函数、内联函数（虽然可以声明但实际不会内联）
8. 重写和隐藏的区别？—— 重写要求函数签名完全一致且基类有 virtual；隐藏只需同名
9. 虚函数的性能影响？—— 一次间接寻址的额外开销，以及无法内联
10. C++11 中 `override` 和 `final` 的作用？—— `override` 显式标记重写，`final` 禁止进一步重写
