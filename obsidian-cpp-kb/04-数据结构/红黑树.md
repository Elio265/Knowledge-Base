# 红黑树

## 一句话理解
红黑树是一种自平衡二叉搜索树，通过节点颜色（红/黑）约束确保最长路径不超过最短路径的两倍，是"近似平衡"的数据结构。

## 核心原理

### 红黑树的五大性质
1. 每个节点要么是红色，要么是黑色
2. 根节点是黑色
3. 所有叶子节点（NIL 空节点）是黑色
4. **红色节点的两个子节点都是黑色**（即不存在连续的红色节点）
5. 从任一节点到其每个叶子节点的所有简单路径上，**黑色节点数量相同**（黑高相等）

这些性质保证了红黑树的平衡性：最长路径（红黑相间）不超过最短路径（全黑）的两倍，因此查找时间复杂度稳定在 O(log n)。

### 红黑树 vs AVL 树
- AVL 树是严格平衡（左右子树高度差 ≤ 1），红黑树是近似平衡
- 红黑树的旋转操作比 AVL 树少，插入/删除性能更优
- C++ STL 中的 `map` 和 `set` 底层使用红黑树实现（而非 AVL 树）

## 底层实现

### 节点定义
```cpp
enum Color { RED, BLACK };

template<class K, class V>
struct RBTreeNode {
    K key;
    V value;
    Color color;
    RBTreeNode* left;
    RBTreeNode* right;
    RBTreeNode* parent;

    RBTreeNode(K k, V v, Color c = RED)
        : key(k), value(v), left(nullptr), right(nullptr), parent(nullptr), color(c) {}
};
```

新插入节点默认为红色，因为插入红色节点不会破坏性质5（黑高相等），代价最小。

### 旋转操作

#### 左旋
```cpp
void rotateLeft(Node* parent) {
    Node* subR = parent->_right;
    Node* subRL = subR->_left;
    // 子节点上位，父节点下沉
    parent->_right = subRL;
    if (subRL) subRL->_parent = parent;
    subR->_left = parent;
    // 更新父指针
    Node* ppnode = parent->_parent;
    parent->_parent = subR;
    if (ppnode == nullptr) _root = subR;
    else if (ppnode->_left == parent) ppnode->_left = subR;
    else ppnode->_right = subR;
    subR->_parent = ppnode;
}
```

#### 右旋
```cpp
void rotateRight(Node* parent) {
    Node* subL = parent->_left;
    Node* subLR = subL->_right;
    parent->_left = subLR;
    if (subLR) subLR->_parent = parent;
    subL->_right = parent;
    Node* ppnode = parent->_parent;
    parent->_parent = subL;
    if (ppnode == nullptr) _root = subL;
    else if (ppnode->_left == parent) ppnode->_left = subL;
    else ppnode->_right = subL;
    subL->_parent = ppnode;
}
```

### 插入操作与修复

插入步骤：
1. 按 BST 规则插入新节点（默认红色）
2. 检查红黑树性质是否被破坏
3. 根据父节点和叔叔节点的情况进行调整

插入修复的几种情况（cur=当前节点，p=父节点，g=祖父节点，u=叔叔节点）：

```
情况 A：p 为黑色 → 直接结束，无需调整
情况 B：p 为红色（破坏性质4），此时需进一步判断 u 的颜色
```

#### 情况 1：叔叔节点 u 为红色
- 将 p 和 u 改为黑色，g 改为红色
- 将 cur 指向 g，向上继续检查

```
     g(黑)              g(红)
    /    \             /    \
  p(红)  u(红)  →    p(黑)  u(黑)
   |
  cur(红)
```

#### 情况 2：叔叔节点 u 为黑色或不存在（需要旋转）
有四种旋转子情况：

| 插入位置 | 场景 | 操作 |
|---------|------|------|
| p 是 g 的左子，cur 是 p 的左子 | 左左（LL） | 对 g 右旋 + 变色（p 黑，g 红） |
| p 是 g 的右子，cur 是 p 的右子 | 右右（RR） | 对 g 左旋 + 变色（p 黑，g 红） |
| p 是 g 的左子，cur 是 p 的右子 | 左右（LR） | 先对 p 左旋，再对 g 右旋 + 变色（cur 黑，g 红） |
| p 是 g 的右子，cur 是 p 的左子 | 右左（RL） | 先对 p 右旋，再对 g 左旋 + 变色（cur 黑，g 红） |

#### 插入修复代码
```cpp
void insertFixup(Node* node) {
    while (node->parent && node->parent->color == RED) {
        Node* parent = node->parent;
        Node* grandparent = parent->parent;
        if (parent == grandparent->_left) {
            Node* uncle = grandparent->_right;
            if (uncle && uncle->color == RED) {
                // Case 1: 叔叔为红 - 变色后向上递归
                parent->color = BLACK;
                uncle->color = BLACK;
                grandparent->color = RED;
                node = grandparent;
            } else {
                // 叔叔为黑或不存在 - 需要旋转
                if (node == parent->_right) {
                    // LR 情况
                    node = parent;
                    rotateLeft(node);
                }
                // LL 情况
                parent->color = BLACK;
                grandparent->color = RED;
                rotateRight(grandparent);
            }
        } else {
            // 对称处理（parent 是 grandparent 的右子）
            Node* uncle = grandparent->_left;
            if (uncle && uncle->color == RED) {
                parent->color = BLACK;
                uncle->color = BLACK;
                grandparent->color = RED;
                node = grandparent;
            } else {
                if (node == parent->_left) {
                    node = parent;
                    rotateRight(node);
                }
                parent->color = BLACK;
                grandparent->color = RED;
                rotateLeft(grandparent);
            }
        }
    }
    _root->color = BLACK; // 强制根为黑色
}
```

### 删除操作与修复

删除步骤：
1. 按 BST 规则删除节点
2. 若删除的是红色节点，直接结束
3. 若删除的是黑色节点，需要通过 `removeFixup` 修复红黑树性质

删除修复的核心思路：给当前节点增加一重"黑色"，然后通过旋转和变色将多余的"黑色"向上传递或消除。

#### 删除修复的四种情况（设当前节点为 x，兄弟节点为 w）
1. **w 为红色**：将 w 变黑，父节点变红，左旋/右旋父节点，更新兄弟节点
2. **w 为黑且 w 的两个子节点均为黑**：将 w 变红，x 指向父节点
3. **w 为黑且 w 的左子为红、右子为黑**：右旋 w，交换 w 与左子颜色，转化为情况4
4. **w 为黑且 w 的右子为红**：将 w 染成父节点色，父节点和 w 右子变黑，左旋父节点，结束

## 常见应用场景
- C++ STL 中 `std::map` / `std::set` 的底层实现
- Java 中 `TreeMap` / `TreeSet` 的底层实现
- Linux 内核进程调度（CFS 调度器中的 red-black tree）
- 文件系统的目录管理
- 内存管理（虚拟内存区域的查找）
- 定时器管理（Nginx 等高性能服务器）

## 容易踩坑的地方
1. **新插入节点必须是红色**：如果插入为黑色，会直接破坏性质5（黑高相等），修复更复杂
2. **根节点必须是黑色**：在调整结束后要强制将根节点设为黑色
3. **插入修复中叔叔节点不存在的处理**：叔叔为空时按黑色处理（对应性质3：空节点为黑色）
4. **删除修复的循环条件**：需要处理 `node == nullptr` 的情况，以及 `node->_parent` 的访问安全性
5. **旋转后父指针的更新**：红黑树有 `parent` 指针，旋转操作中必须正确维护所有父指针关系
